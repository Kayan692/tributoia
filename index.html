Passo 1 — Criar o Webhook no Bitrix24

O Webhook é uma chave de acesso que permite ler os dados do CRM sem expor sua senha. Siga o passo a passo abaixo.

📍 Como gerar o Webhook Token

  1. Acesse seu Bitrix24 como administrador
  2. No menu lateral, clique em Aplicativos (ou Marketplace)
  3. Clique em Webhooks → Adicionar webhook de entrada
  4. Em "Permissões", marque: CRM (crm) — somente leitura já é suficiente
  5. Clique em Salvar
  6. Copie a URL gerada. Ela terá o formato:
    https://suaempresa.bitrix24.com.br/rest/1/abc123xyz.../
  7. A URL do seu Bitrix24 é a parte antes de /rest/
    O Token é a parte após o número do usuário (ex: abc123xyz...)
💡 Caminho alternativo: Configurações → Integrações → REST API → Adicionar webhook

🔒 É seguro?

  • O token dá acesso somente leitura ao CRM
  • Ele não é sua senha — você pode revogar a qualquer momento
  • Este sistema nunca envia seus dados para terceiros
  • Tudo roda no seu próprio computador ou servidor

Passo 4 — Publicar Online (link para o time)

Para que todo o time acesse o dashboard pelo navegador, você precisa hospedar este arquivo online. Veja as opções abaixo — todas gratuitas.

🚀 Opção A — Netlify Drop (mais fácil, 2 minutos)

  1. Acesse app.netlify.com/drop
  2. Arraste e solte este arquivo HTML na página
  3. Em segundos você recebe um link como https://amazing-biscuit-123.netlify.app
  4. Compartilhe o link com o time
  5. Para atualizar: salve as configurações (botão abaixo) e faça o drop novamente
⚠️ Atenção: o Netlify Drop é público. Qualquer pessoa com o link pode acessar. Para proteger com senha, use a opção B.

☁️ Opção B — Vercel (com proteção por senha)

  1. Instale o Node.js em nodejs.org
  2. Abra o terminal (cmd no Windows) e execute:
    # Instalar Vercel CLI npm install -g vercel # Fazer deploy deste arquivo vercel sales_intel_completo.html --name sales-intel
  3. Faça login com GitHub quando solicitado
  4. Você receberá um link https://sales-intel.vercel.app

💻 Opção C — Rodar local (sem internet necessária)

  1. Salve este arquivo em qualquer pasta
  2. Abra com o Chrome ou Edge
  3. O sistema funciona 100% offline — os dados vêm do seu Bitrix24 diretamente
  4. Para o time acessar na mesma rede Wi-Fi, use o Python:
    python -m http.server 8080
    Acesse: http://[SEU-IP]:8080/sales_intel_completo.html
✅ Suas configurações serão salvas dentro deste arquivo HTML quando você clicar em "Salvar e Entrar". Faça o deploy depois de clicar em salvar para que o link já venha configurado.

🤖 Análise por IA — Especialista Comercial B2B

Clique em "Gerar Análise" para que a IA leia os dados do seu CRM e entregue insights priorizados como um analista comercial B2B sênior.

📋 Roteiro de Daily — Como usar este sistema
1
0–3 min
Briefing da IA
Leia o parágrafo de briefing gerado acima. Contextualiza o estado da operação sem precisar olhar número por número. Se o time não tem Claude API, leia os alertas críticos da aba Daily em voz alta.
2
3–8 min
Alertas críticos
Aba "Daily" → seção Alertas Críticos. Para cada um: "Fulano, o deal X está sem atividade há N dias. Próximo passo?" — 30 segundos por resposta. Sem monólogos.
3
8–18 min
Pipeline prioritário
Aba "Pipeline" — os deals vermelhos e laranjas no topo são os que precisam de ação hoje. Para cada um: probabilidade real de fechar? Próxima ação, quem faz, quando?
4
18–23 min
Produtividade ontem
Tabela de produtividade: quem está abaixo de 50% da média? Não é julgamento — é diagnóstico. Bloqueio técnico, prospect difícil ou desorganização? O gestor anota e resolve em 1:1.
5
23–25 min
Commit de ações
3–5 ações com dono e prazo. "Hoje 14h: Lidia contata deals X e Y. Amanhã 9h: Victor atualiza motivos de perda." Verificadas na daily seguinte — sem commit, sem cobrança.

🚀 Como publicar online (link para o time)

Escolha a opção que faz mais sentido para você. Netlify Drop é a mais rápida.

Opção A — Netlify Drop (2 minutos, gratuito)

  1. Clique em "Baixar este arquivo configurado" abaixo
  2. Acesse app.netlify.com/drop
  3. Arraste o arquivo HTML baixado para a área da página
  4. Em <60 segundos você recebe um link público
  5. Compartilhe com o time — cada pessoa abre no próprio navegador

Opção B — GitHub Pages (grátis e permanente)

  1. Crie uma conta em github.com (gratuito)
  2. Crie um novo repositório público
  3. Faça upload do arquivo HTML (renomeie para index.html)
  4. Vá em Settings → Pages → Source: main branch
  5. Link: https://seuuser.github.io/nomedorepositorio

Opção C — Rodar local na rede do escritório

# Abrir terminal na pasta do arquivo # Windows: botão direito na pasta → "Abrir terminal" python -m http.server 8080 # ou, se não tiver Python: npx serve . # Acesse no computador: http://localhost:8080/sales_intel_completo.html # Acesse de outros computadores na mesma rede: http://[SEU-IP-LOCAL]:8080/sales_intel_completo.html # (Veja seu IP em: ipconfig no Windows, ifconfig no Mac/Linux)

🔄 Como atualizar quando o sistema receber melhorias

  1. Baixe a nova versão do arquivo HTML
  2. Antes de substituir, vá em Config → Exportar configurações e salve o JSON
  3. Substitua o arquivo antigo pelo novo
  4. Em Config → Importar configurações, cole o JSON salvo
  5. Pronto — todas as suas configurações estão de volta
